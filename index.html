DVB-C+ & Holo+ OS: Gesamtspezifikation

Das Betriebssystem Holo+ bildet die native Schnittstellenplattform für das hybride Kabelsystem DVB-C+. Es regelt die hardwarebeschleunigte Dekodierung hochbitratiger Medienströme, die Ausführung von sendereigenen interaktiven Overlays (OmniView) sowie globalen Systemapplikationen (Apps).

group Werde Teil der TV-Revolution!

DVB-C+ und Holo+ OS sind Open-Source-Projekte, die von der Community leben. Egal ob du Kernel-Entwickler bist, dich mit Tuner-Architekturen auskennst oder App-Blueprints schreibst.

gavel 1. Die Holo+ Werbe- & Performance-Charta

  • Erlaubt (Klassische TV-Werbung): Traditionelle Werbeblöcke im linearen Videostream der Sender sind vollumfänglich erlaubt.
  • Erlaubt (Moderate App- & OmniView-Werbung): Innerhalb globaler Apps (.hap) und dem OmniView-Interface (.oip) ist Werbung in moderaten Mengen gestattet.
  • Strengstens verboten (Tracker & Werbe-Spam): Jegliche Form von unsichtbaren Trackern zur Datensammlung ist verboten.
  • Kein AI-Slop: Holo+ verzichtet vollständig auf künstlich aufgeblasene Marketing-Features.

router 2. Hardware- & Tuner-Architektur & Retro-Upscaler

  • Single-Tuner-Geräte: Exklusiv für den DVB-C+ Standard.
  • Dual-Tuner-Geräte (Retail-Standard): Bieten volle Abwärtskompatibilität.
  • Der Native Retro-Upscaler: Für analoge Signale und digitale Streams mit einer Auflösung von 480p und weniger.

    info Funktionsweise: Nutzt einen präzisen Integer-Scaling-Algorithmus.

    warning Hardware-Voraussetzung: Dieses Feature erfordert ausreichende GPU-Power.

enhanced_encryption 3. In-Stream Keystore-Update & Offline-Sicherheit

  • Globales Keystore-Karussell: Jeder DVB-C+ Transponder strahlt kontinuierlich den Master-Keystore aus.
  • Mathematische Prüfziffer (Fallback): Offizielle Zertifikate enthalten eine integrierte Prüfziffer.
  • 30-Tage-Vertrauensfenster: Schlägt die Formel fehl, läuft die Software für 30 Tage als Kulanz.

badge 4. Sicherheits- & Zertifikats-Struktur

Zertifikate sind UTF-8-kodierte Textdateien, die das OS in numerische Werte auflöst:

shield_with_heart .hoc

256 Zeichen. Für sendergebundene .oip-Interfaces.

token .hac

256 Zeichen. Für globale Apps (.hap).

admin_panel_settings .hsc

512 Zeichen. Zwingend erforderlich für System-Services.

Syntax-Aufbau einer Zertifikatsdatei

comment#Holo+ Application/OmniView/System Certificate
comment#Distributed by NeoAudio on the 1st of June 2026 (gültig 4 Jahre)
comment#Certificate Key:
start¡[256/512 Zeichen Schlüssel]!output
comment#Checksum:
start¡[32 Zeichen Zahl, mathematische Formel]!output

terminal Parser-Mechanik: Das invertierte Ausrufezeichen ¡ dient dem OS-Kernel als absoluter Startpunkt.

report_problem Der Development-Modus (Unsigniert)

Software ohne gültiges Zertifikat läuft in einer Sandbox ohne Systemzugriff.

apps 6. Globale Apps & Kernel-API Struktur (.hap)

label Paket-Namensschema: com.[Entwickler].[Typ].[Appname]_[Version]_[TargetDevice].hap

Macht Unterscheidungen zwischen drei strukturellen Abhängigkeitstypen:

  1. Shared System Frameworks: Globale Core-Frameworks (react, enact).
  2. System Core APIs (holo-app-core): Restriktive System-Schnittstellen.
  3. Selbstgeladene Dependencies: Eigene Libraries im Paket-Pfad.

Blueprint: Struktur einer Holo+ App (.hap)

inventory_2 com.developer.type.appname_1.0.0_universal.hap
schema appinfo.json <- Das App-Manifest
token certificate.hac <- Das App-Zertifikat
admin_panel_settings system_auth.hsc <- Das System-Zertifikat
html main.html <- Der App-Startpunkt
folder_open deps <- Ordner für Abhängigkeiten
javascript custom-slider.js <- Entwickler-Dependency
folder_shared dep-apps <- Erforderliche Dependency-Apps
folder_open media_assets <- Lokale Medien assets
